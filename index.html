Constraint Architecture — 8 Business Rules

C1
Fully InvestedAll weights sum to exactly 1.0 — no cash drag permitted.
C2
Position Floor & Cap (Linking)Binary Y + continuous X: 5% min if selected, 40% max. Enforced via MINLP linking constraints.
C3
Cardinality MaximumNo more than 10 stocks held simultaneously — controls transaction cost and management overhead.
C4
Cardinality MinimumAt least 2 stocks required — prevents degenerate single-stock "portfolios".
C5
Sector Cap (Dynamic)No single sector exceeds 40%. Sector groups built live from yfinance metadata — no hardcoding.
C6
Defensive FloorMust hold ≥1 Healthcare or Consumer Defensive stock. Ensures downside protection in any allocation.
C7
Tech → Financials ContingencyIf any Technology stock is held, must also hold a Financials stock. Macro credit-cycle hedge.
C8
XOM / CVX Mutual ExclusionCannot hold both simultaneously — they share the same macro factor exposure with no diversification benefit.

Production Architecture

⚡
Live Data Pipeline
GitHub Actions cron fires at 3:55 PM ET (Mon–Fri). Captures 1-min bar for 36 tickers via yfinance, deduplicates, checks holidays, appends to CSV, and auto-commits — zero manual steps.
GITHUB ACTIONS · CRON
🧮
MINLP Solver
Bonmin (COIN-OR) via IDAES. Mixed-integer nonlinear programming handles binary stock selection and quadratic risk simultaneously. Sweeps 28 risk levels to trace the complete frontier.
BONMIN · PYOMO · IDAES
📡
Dynamic Constraints
All sector constraints built from live yfinance metadata at runtime — no hardcoded mappings. Add or remove a ticker and the constraint architecture updates automatically.
YFINANCE · DYNAMIC SECTORS
🔍
Pre-Optimization Screening
Two-stage filter before any solve: 14-day Wilder RSI (30–70 band) removes trend-failing and overbought names; mean return floor removes chronic underperformers.
RSI · RETURN FILTER
🔄
Sliding Window Backtest
12-month rolling training windows, 11 out-of-sample test periods. Each window re-runs the full MINLP to pick next-month weights from prior data — true out-of-sample validation.
12-MONTH WINDOW · 11 PERIODS
✅
Constraint Validation
Post-solve verification suite checks all 8 constraint categories across every frontier solution. Zero violations reported across all 28 MINLP solutions.
